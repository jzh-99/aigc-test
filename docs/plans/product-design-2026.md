## 1. 产品定位与用户分层

### 1.1 产品定位

一个以 AI 为核心驱动力的多媒体内容创作平台。用户可以通过对话、流程引导、或自由画布三种方式，完成从创意到成片/成图的全链路创作。

平台不是工具集合，而是**创作伙伴**——AI 理解用户意图，主动推进创作流程，用户在关键节点做决策和审美判断。

### 1.2 用户分层

#### 用户 A：小白自由创作者
- **典型需求**：做一组产品图、把照片 P 成油画风、参考这张图做一张海报、生成一段 BGM
- **特征**：不了解 AI 参数，不关心工作流，只关心结果；需要即时反馈；容易被复杂界面劝退
- **核心诉求**：说一句话，AI 帮我搞定，结果直接看到
- **对应模式**：对话模式

#### 用户 B：视频制作者
- **典型需求**：制作一条 30-120 秒的 AI 短视频，有完整的剧本-分镜-角色-配音-合成流程
- **特征**：有一定创作经验，理解视频制作流程；需要对每一步结果做审核和调整；对质量要求高
- **核心诉求**：流程清晰，每步结果可见可改，不走弯路
- **对应模式**：步骤模式

#### 用户 C：专业 AIGC 创作者
- **典型需求**：精细控制生成参数、搭建复杂的多节点工作流、复用和组合已有素材
- **特征**：熟悉 AI 生成原理，有明确的工作流思维；需要细粒度控制；可能同时管理多个项目
- **核心诉求**：自由、精确、高效
- **对应模式**：画布模式

#### 用户 D：企业批量创作
- **典型需求**：批量生成 100 款产品海报、为不同城市生成定制化广告素材
- **特征**：有固定模板，需要变量替换；关注效率和成本；可能需要团队协作审核
- **核心诉求**：一次配置，批量产出，结果可管理
- **对应模式**：画布模式的批量视图（中长期）

## 2. 核心设计理念

### 2.1 一个底层，三种界面

三种模式共用同一套底层数据结构（canvas nodes + edges）和执行引擎。模式切换是**视图层的变化**，不是数据层的变化。

这意味着：
- 用户在步骤模式里生成的角色图，可以切换到画布模式后继续手动调整连线
- 对话模式里 agent 创建的节点，在画布模式里完全可见可编辑
- 不同模式之间的切换不丢失任何数据

### 2.2 AI 是推进者，用户是决策者

AI 负责：理解意图、规划步骤、执行生成、处理参数细节
用户负责：审美判断、内容决策、方向调整

界面设计要体现这个分工——AI 的操作要可见、可撤销、可接管。用户随时可以从 AI 引导模式切换到手动模式。

### 2.3 积分感知：透明且需确认

任何消耗积分的生成操作，**执行前必须经过用户确认**。确认不是弹窗打断，而是 agent 在对话流中展示一张任务确认卡片，用户看清楚再决定。

**确认卡片样式（三种模式通用）：**
```
┌─────────────────────────────────────────────────┐
│  准备执行以下任务                                  │
│                                                  │
│  • 生成角色图 × 3 张          约 15 积分           │
│  • 生成场景图 × 2 张          约 10 积分           │
│  • 生成视频片段 × 5 段        约 150 积分          │
│                                                  │
│  预估总消耗  约 175 积分   当前余额  320 积分        │
│                                                  │
│  [取消]                    [确认执行]             │
└─────────────────────────────────────────────────┘
```

**各模式下的触发时机：**

- **对话模式**：agent 规划好 skill 后，先展示确认卡片，用户点"确认执行"后才开始生成。参数仍由 agent 自动决定，卡片里展示任务摘要而非参数细节。
- **步骤模式**：每个步骤的"生成"按钮点击后，展示本步骤的确认卡片（而非直接执行）。用户在卡片里可以看到本步骤的任务数量和预估消耗。
- **画布模式**：节点执行按钮点击后，展示单节点的确认卡片。批量执行多节点时，汇总展示。

**边界情况处理：**
- 余额不足：确认按钮禁用，卡片底部显示"余额不足，差 X 积分"+ 充值链接
- 取消后：任务节点保留，状态回到"待执行"，用户可修改后重新触发
- 预估用"约"，实际扣费以执行结果为准，失败自动退还
- 单次消耗极小（如纯文本生成、剧本生成）：可豁免确认，直接执行

### 2.4 结果优先，过程收起

无论哪种模式，生成结果（图片、视频、文本）都应该是界面的主角，占据最大的视觉空间。参数配置、节点连线、执行日志都是次要信息，默认收起或缩小。

### 2.4 进入即创作

用户访问 `/canvas` 时直接进入最近的画布，没有历史时展示引导页。画布切换通过 header 下拉完成。大厅页面（`/canvas`）保留，用于有项目管理需求的用户浏览和整理所有画布，但不作为默认落点。

## 3. 三种创作模式详细设计

### 3.1 对话模式（小白自由创作）

#### 3.1.1 整体布局

```
┌─────────────────────────────────────────────────────┐
│  header: logo  |  [画布名/切换▼]  |  [切换到画布模式] │
├─────────────────────────────────────────────────────┤
│                                                     │
│              对话区（全屏）                           │
│                                                     │
│   ┌─────────────────────────────────────────────┐   │
│   │  AI: 你好！我可以帮你生成图片、视频、海报…    │   │
│   │      你想做什么？                            │   │
│   └─────────────────────────────────────────────┘   │
│                                                     │
│   ┌─────────────────────────────────────────────┐   │
│   │  用户: 帮我把这张照片做成油画风格             │   │
│   │  [📎 photo.jpg]                             │   │
│   └─────────────────────────────────────────────┘   │
│                                                     │
│   ┌─────────────────────────────────────────────┐   │
│   │  AI: 好的，我来处理…                         │   │
│   │  ┌──────────┐  ┌──────────┐  ┌──────────┐  │   │
│   │  │  [图片1]  │  │  [图片2]  │  │  [图片3]  │  │   │
│   │  └──────────┘  └──────────┘  └──────────┘  │   │
│   │  满意吗？可以告诉我调整方向                   │   │
│   └─────────────────────────────────────────────┘   │
│                                                     │
├─────────────────────────────────────────────────────┤
│  [📎上传]  [输入框：说说你想做什么…]  [发送]          │
└─────────────────────────────────────────────────────┘
```

画布完全隐藏。用户感知不到节点和连线的存在。

#### 3.1.2 Agent 能力范围（对话模式）

对话模式的 agent 需要支持以下 skill，每个 skill 对应一组画布节点操作，但对用户透明。**所有参数由 agent 根据上下文自动决定，用户不需要手动选择**：

| 用户说的话 | Agent 识别的 skill | 底层操作 | Agent 自动决定的参数 |
|---|---|---|---|
| 帮我生成一张 xxx 的图 | `image_gen` | 创建 text_input + image_gen 节点并执行 | 分辨率 2k、画幅 1:1、数量 3 |
| 参考这张图，做一张 xxx 风格的 | `image_gen_with_ref` | 创建 asset + text_input + image_gen 节点 | 同上 |
| 把这张图 P 成 xxx 风格 | `image_style_transfer` | 同上，prompt 强调风格迁移 | 同上 |
| 做一组 xxx 的图（N 张） | `image_batch` | 创建多个并行 image_gen 节点 | 数量取用户指定值 |
| 帮我做一张海报，内容是 xxx | `poster_gen` | 创建 text_input + image_gen，prompt 模板化为海报构图 | 分辨率 2k、画幅 2:3 |
| 生成一段视频，内容是 xxx | `video_gen` | 创建 text_input + video_gen 节点 | 时长 5s、画幅 16:9 |
| 参考这个视频的风格，生成 xxx | `video_gen_with_ref` | 创建 asset + text_input + video_gen | 同上 |
| 帮我配一段 BGM | `bgm_gen` | 创建 audio_gen 节点（待实现） | 时长与视频等长 |
| 帮我做 AI 配音 | `tts_gen` | 创建 tts 节点（待实现） | 语速正常、音色默认 |

#### 3.1.3 结果展示（内联渲染）

生成结果直接内嵌在对话气泡里，用户全程不需要打开画布：
- 图片：最多 3 列网格，点击放大，右键/长按下载
- 视频：内嵌播放器，自动静音循环
- 文本：代码块样式，支持复制

每个结果气泡底部有操作栏：
```
[再生成一次]  [调整参数]  [保存到素材库]  [继续用这个]
```

"继续用这个"会把当前结果作为下一步的参考输入，agent 询问下一步意图。

**内联渲染的实现机制：**

agent panel 在轮询节点状态时，一旦检测到节点状态变为 `completed`，就向对话消息列表注入一条 `type: 'result'` 的消息，内容包含节点输出的图片/视频 URL 列表。这条消息由前端自动插入，不经过 LLM，延迟极低。

消息类型扩展：
```ts
type ChatMessage =
  | { role: 'user' | 'assistant'; content: string }
  | { role: 'result'; nodeId: string; outputs: Array<{ url: string; type: 'image' | 'video' }> }
  | { role: 'thinking'; content: string }   // agent 规划中的中间状态
```

#### 3.1.4 Agent 默认行为：逐步引导

**默认模式是逐步引导，不是自动完成。**

agent 规划好工作流后，默认逐步推进，每一步都等用户确认再继续：

```
用户：帮我做一个民国风格的短视频

Agent：好的，我来帮你规划。先从剧本开始——
       你希望故事讲什么？有没有特定的人物或情节？

用户：一个茶馆里的邂逅，两个陌生人

Agent：好的，我来生成剧本…
       [剧本全文展示]
       满意吗？可以直接修改，或者告诉我调整方向。
       满意的话回复"好"，我们进入下一步：分镜拆分。

用户：好

Agent：正在拆分分镜…
       [分镜列表展示]
       共 6 个镜头，你可以点击任意镜头修改文案。
       确认后我们生成角色和场景图。
```

每一步结果都内联展示，用户看清楚再推进。这样用户对整个创作过程有掌控感，也更容易在某一步叫停修改。

**端到端自动模式（用户主动触发）：**

当用户明确表达"直接做完"、"一键生成"、"帮我全部搞定"等意图时，agent 切换到自动模式，一次性规划并执行所有步骤，中间只在积分确认卡片处暂停：

```
用户：帮我做一个民国风格的30秒短视频，直接帮我全部做完

Agent：好的，我来一次性规划所有步骤。

       ┌─────────────────────────────────────────────┐
       │  准备执行以下任务                              │
       │  • 生成角色图 × 3 张          约 15 积分      │
       │  • 生成场景图 × 2 张          约 10 积分      │
       │  • 生成视频片段 × 5 段        约 150 积分     │
       │  预估总消耗  约 175 积分   当前余额  320 积分  │
       │  [取消]                    [确认执行]         │
       └─────────────────────────────────────────────┘

用户：[点击确认执行]

Agent：正在生成角色图…
       ┌──────────────────────────────────────────┐
       │  [角色A-正面]  [角色A-侧面]  [角色A-背面]  │
       └──────────────────────────────────────────┘
       正在生成场景图…
       ┌──────────────────────────────────────────┐
       │  [茶馆内景]  [街道外景]                   │
       └──────────────────────────────────────────┘
       正在生成视频片段…（5/5 完成）
       ┌──────────────────────────────────────────┐
       │  [片段1 ▶]  [片段2 ▶]  [片段3 ▶]  …     │
       └──────────────────────────────────────────┘
       全部完成！你觉得效果怎么样？
       如果想调整某个角色或场景，直接告诉我。
```

**模式识别关键词（agent 侧）：**
- 触发自动模式："直接做完"、"一键生成"、"全部帮我搞定"、"不用问我"、"自动完成"
- 保持逐步模式：其他所有情况（默认）

**迭代修改（两种模式通用）：**

用户说"把角色A改成老年男性"时，agent 定位到对应的 text_input 节点，修改 `config.text`（imagePrompt），重新执行该节点及其下游 image_gen 节点，结果再次内联展示。这复用了现有的 `apply_workflow` + `guide_step` 机制，不需要新的指令类型。

#### 3.1.4 上传交互

输入框左侧的 📎 按钮支持：
- 拖拽文件到对话区
- 点击选择本地文件
- 粘贴图片（Ctrl+V）
- 从素材库选择（弹出素材库 picker）

上传后文件以缩略图形式附在输入框上方，可以删除。发送后 agent 自动识别文件类型并规划对应的 skill。

#### 3.1.5 模式切换入口

header 右上角有"切换到画布模式"按钮。点击后：
- 当前对话中 agent 创建的所有节点在画布上可见
- 右侧 agent 面板保留，但变为辅助角色
- 用户可以手动拖拽、连线、调整参数

#### 3.1.6 新建对话

header 中间的画布名称旁有下拉箭头，展开后：
```
┌─────────────────────┐
│ ✦ 新建对话           │
├─────────────────────┤
│ 油画风格照片  今天    │
│ 产品海报 x5  昨天    │
│ 短视频-咖啡馆  4月18 │
│ ...                 │
├─────────────────────┤
│ 查看全部画布          │
└─────────────────────┘
```

"新建对话"直接创建新 canvas 并跳转，不需要先进大厅选择。

#### 3.1.7 画布作为高级视图

对话模式下，画布始终存在于后台，但对小白用户不可见。header 右上角有一个低调的"查看画布"入口（图标按钮，不是主要操作），供好奇的用户或需要精细调整的用户打开。

打开画布后：
- agent 创建的所有节点在画布上可见，按创建顺序排列
- 右侧 agent 面板保留，用户可以继续对话
- 用户可以手动调整节点参数、重新连线
- 关闭画布（点击"返回对话"）后，对话历史和节点状态都保留

**设计原则**：画布是对话的底层实现，不是对话的替代品。小白用户不需要知道画布的存在，专业用户可以随时打开它做精细控制。（视频流程化创作）

#### 3.2.1 整体布局

```
┌──────────────────────────────────────────────────────────────────┐
│  header: [← 返回]  项目名称  |  [步骤模式]  [画布模式]  |  [保存] │
├────────────────┬─────────────────────────────────────────────────┤
│                │                                                  │
│  步骤导航栏     │              主内容区                            │
│  (240px)       │                                                  │
│                │                                                  │
│  ● 剧本        │                                                  │
│  ○ 分镜        │                                                  │
│  ○ 角色 & 场景  │                                                  │
│  ○ 配音 & BGM  │                                                  │
│  ○ 视频合成     │                                                  │
│                │                                                  │

│  [AI 助手]     │                                                  │
│                │                                                  │
└────────────────┴─────────────────────────────────────────────────┘
```

左侧步骤导航固定，主内容区根据当前步骤渲染对应的全屏内容。

#### 3.2.2 步骤导航栏

每个步骤有三种状态：
- `待完成`：灰色圆点，不可点击（依赖上一步完成）
- `进行中`：蓝色圆点 + 动画，当前步骤
- `已完成`：绿色勾，可点击回看/修改

步骤之间有连线，已完成的连线变绿。

导航栏底部有"AI 助手"折叠面板，展开后是一个小型对话框，用于在步骤内提问或请求 AI 帮助（如"帮我改一下这段剧本"）。

**主内容区布局：**

```
│  ┌────────────────────┐  │  │  剧本全文                       │  │
│  │ 一个咖啡馆里的邂逅…  │  │  │                                │  │
│  └────────────────────┘  │  │  第一幕：清晨，咖啡馆门口…       │  │
│                          │  │  角色A走进咖啡馆，点了一杯…      │  │
│  风格                     │  │  ...                           │  │
```

"下一步"按钮在剧本生成完成后激活。

#### 3.2.4 Step 2：分镜

**主内容区布局：**
```
┌─────────────────────────────────────────────────────────────────┐
│  Step 2 / 5  分镜拆分                                            │
```

## 4. 入口与导航设计

### 4.1 `/canvas` 路由逻辑

```
用户访问 /canvas
    ↓
有历史画布？
    ├── 是 → 跳转到最近编辑的画布 /canvas/editor/:lastId
    └── 否 → 展示引导页（全屏）
              ┌─────────────────────────────────┐
              │  你想做什么？                    │
              │                                 │
              │  [🎬 制作视频]  [🖼️ 生成图片]   │
              │  [🎨 自由创作]  [📦 批量生产]   │
              └─────────────────────────────────┘
```

四个入口对应：
- **制作视频** → 创建新画布，进入步骤模式，模板预置 script_writer + storyboard_splitter 节点
- **生成图片** → 创建新画布，进入对话模式，agent 预置"你想生成什么图片？"
- **自由创作** → 创建新画布，进入画布模式，空白画布
- **批量生产** → 中长期功能，暂时跳转到 `/generation` 页面

大厅页面（`/canvas`）保留，用于有项目管理需求的用户浏览、搜索、整理所有画布。入口在 header 下拉的"查看全部画布"，以及侧边栏导航（可选）。大厅不是默认落点，但始终可达——用户直接访问 `/canvas` 且有历史画布时会被重定向到最近画布，可通过 header 下拉"查看全部画布"回到大厅列表。

大厅页面在现有基础上补充：
- 搜索框（按名称过滤）
- 模式标签筛选（视频项目 / 对话 / 画布）
- 排序（最近编辑 / 创建时间 / 名称）
- 右键菜单：重命名、复制、删除、移动到工作区

## 8. 开发规划

### 优先级原则

1. **先打通核心用户路径**，再做体验优化
2. **步骤模式优先于对话模式**：步骤模式的 UI 框架更清晰，且直接服务于视频制作者这个高价值用户群
3. **对话模式的 agent 能力优先于 UI 重构**：先让 agent 能处理更多 skill，再改界面
4. **新节点类型（audio/tts）是步骤模式的前置依赖**，需要提前排期

### Phase 2：对话模式重构（预计 2-3 周）

**目标**：小白用户可以通过纯对话完成图片生成、风格迁移、海报制作等需求，不需要接触画布。

#### Week 5：对话模式 UI
- [ ] 对话模式布局（全屏对话，画布隐藏）
- [ ] **结果内联渲染**（核心）：轮询到节点 `completed` 时，自动向对话注入 `type: 'result'` 消息，渲染图片网格/视频播放器
- [ ] 结果操作栏（再生成、调整参数、保存、继续用这个）
- [ ] 图片粘贴上传（Ctrl+V）
- [ ] 素材库 picker
- [ ] "查看画布"入口（header 低调图标按钮）

> **内联渲染优先级说明**：这是对话模式能否真正服务小白的关键。当前 guide_step 引导用户去画布点执行，结果也在画布节点里看，小白需要理解"节点"概念才能找到结果。内联渲染把结果带回对话，消除这个认知门槛。实现成本约 80-100 行（扩展消息类型 + 轮询回调注入），应作为 Week 5 第一个任务。

| 维度 | 对话模式 | 步骤模式 | 画布模式 |
|---|---|---|---|
